---
title: Declaring Variables
---

<Callout type="warn">Re-declaration of a variable in the same scope is not allowed.</Callout>

## Using the `var` keyword

```go
var varName type = value
```

> Variables declared without an explicit initial value receive their zero value.

- Either `type`, `value`, or both must be specified:

  ```go
  var name string = "John"  // explicit type and value
  var surname = "Doe"       // type inferred as string
  var age int16             // explicit type, zero value (0)
  ```

- Declaring multiple variables on single line, same or different types:

  ```go
  var name, age = "John", 30  // mixed types, types inferred
  var a, b, c int = 1, 2, 3   // all must be of the declared type
  var x, y int                // both zero value (0)
  ```

- Grouped declaration:

  ```go
  var (
  	name    string = "John"
  	surname        = "Doe"  // type inferred as string
  	age     int             // zero value (0)
  )
  ```

## Using the `:=` syntax

```go
varName := value
```

<Callout type="warn">Only valid inside functions, not at package level.</Callout>

- Multiple variables on single line:

  ```go
  name, age := "John", 30
  ```

  <Callout>

  Re-declaration of a variable in the same scope is not allowed.

  BUT

  If at least one identifier on the left side is new, this syntax is allowed.

  ```go
  x := 10

  x := 20 // <-- compiler: no new variables on left side of :=

  x, y := 20, 30 // y is new, so this is allowed

  fmt.Println(x, y) // 20 30
  ```

  New identifiers are declared and existing ones are reassigned.

  </Callout>

## Using the `const` keyword

Constants are fixed values and cannot change after declaration.

```go
const varName type = value
```

<Callout type="warn">
  Constants must be known at compile time. They cannot depend on variables, function calls, or any other runtime values.
  Expressions that the compiler can evaluate during compilation are allowed. Constants are limited to booleans, numbers,
  and strings.
</Callout>

- Constants may omit the type but must have a value:

  ```go
  const daysInWeek int = 7  // explicit type and value
  const hoursInDay = 24     // type is inferred as int
  ```

- Grouped declaration:

  ```go
  const (
  	daysInWeek = 7
  	hoursInDay = 24
  )
  ```
